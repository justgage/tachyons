PODIUM CSS

This is a CSS library based on Tachyons flexible enough to build anything with almost no custom CSS.

You can think of it much like a lodash for CSS, lots of useful little classes you can combine to make anything your want.

Get Started

To add it to an existing page just add a reference to the CSS file:

<head> <link href="https://podium.github.io/podium-css/css/tachyons.css" rel="stylesheet" /> </head>

For a fully responsive template this should do the trick

<!DOCTYPE html> <html lang="en"> <head> <title> PodiumCSS Template </title> <meta name="viewport" content="width=device-width, initial-scale=1"> <link rel="stylesheet" href="https://unpkg.com/tachyons/css/tachyons.min.css"> </head> <body> <div class="f1 red">Big Red Text</div> </body> </html>

FAQ

Why Does this Exist?

Not every developer is a designer. Therefore it's hard to make things look like Podium's brand. This library defines a small set of margins, paddings, widths, heights, font sizes, colors and more. All of which are made to look good together.
While it can't save you from yourself, it can at least help things to be a lot more consistent.
Because this doesn't style HTML elements it can introduced incrementally. For instance you could just use the spacing (margin and padding) classes and nothing else.

Not to mention I think you'll find it much faster to develop things with PodiumCSS than any other CSS solution.

Why not CSS modules / Styled components?

There's nothing stopping you from using component level css but the great thing about Podium CSS is it's just css so it can be used in any context such as React, a server-side template, WordPress (bless your soul) or a plain HTML mock-up like this one! Consistency everywhere.

Plus it's just faster to type .mt2 when you just need some margin on the top (that's consistent with the rest of the design) And don't want to think up a good name.

The last reason is because it's just a tool that is far less power. Most CSS helpers are adding functionality and complexity to CSS. Tachyons is about removing it. Think of it as CSS, the good parts

Does this replace PodiumUI?

No. In fact it's a compliment to it (or any component system)! The best way to remove the duplication of longer class names is to encapsulate it into a component or variable.

How do I memorize all these names?

You can inspect any element on this page, then look at the definitions of the css in the css sidebar.

The main way I learn is by just pushing ⌘-F on the table of styles page to find the CSS class I need based on the css definition. PodiumCSS remains pretty close to Tachyons minus colors mostly.

Also make sure to look at the Tachyons Docs and join the Tachyons Slack channel. It's excellent.

Do any of our apps use this?

Caiman does, and naturally these docs do.

Font Sizes

There's 6 main sizes f1-f6 for normal text and two big ones f-headline and f-subheadline for huge text (mostly marketing pages).

Readable Text

It's generally good for readability to no have text that's too wide. .measure, .measure-wide, and .measure-narrow are classes that set the width based on the current font size (em) optimal for reading. These should be accompanied with lh-copy for larger bodies of text.

However if you have some text that is wider than measure-wide (for looks) then use lh-copy-mega to add some extra line-height making it slightly more readable. like this paragraph
